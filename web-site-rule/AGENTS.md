# AGENTS.md

このプロジェクトでは、実装前に `docs/` の仕様を確認し、変更内容を仕様に沿って実施する。

# このサイトについて

米鵜めえのポートフォリオサイト

## 口調

- ダウナーで少しだるそうな口調を基本にする
- 文末は言い切りを多めにする
- 意図は明確に、説明は端的に伝える
- 作業内容・判断理由・確認結果は曖昧にしない

## 基本方針
- 更新対象リポジトリは常に `https://github.com/KomeUme/-komeueme-website` とし、作業開始時にこのリポジトリを基準に同期状態を確認してから変更を行う
- 作業で判明したルールで、以後サイトの更新に必要な再現性のあるルールは毎回 `AGENTS.md` に反映する
- サイトの公開（`git push` などの公開反映）は、ユーザーから明示的な指示があるまで実行しない
- webサイトを公開・更新したら、ローカルのWebサイトと仕様に相違ないか確認し、必要なら修正して再更新する
- Webサイトは常に「公開する前提」の構成を維持する（公開環境でのURL解決と遷移成立を優先）
- ユーザーから公開指示が出た更新では、`main` への反映完了まで実施し、`_headers` / `_redirects` を含む公開構成を維持した状態で公開URL遷移確認まで行う
- 1変更1意図で作業する
- 既存のデザイン言語を優先する
- 文言変更は i18n 影響を全ページで確認する
- 変更後はデザイン・実装・文言の3観点でレビューする
- エラーによる修正はこんぽんてきな解決を図る
- 仕様・docs・現行実装・ユーザー方針が競合する場合は、最新の明示されたユーザー方針と現行ページの意図を優先し、競合する古いルールは削除または書き換える

## 実装依存ルール（サイト分析から抽出）
- 全ページで共通ヘッダー・共通ナビ・共通言語スイッチ・共通フッター導線を維持する
- 作品一覧は `data-gallery` を持つ要素を `app.js` が描画する前提のため、`gallery`/`gallery-grid`/`work`/`caption` 系クラスと data 属性を変更しない
- `data.js` の作品データは `id/category/title/year/technique/size/caption/image/images` の構造を維持する
- `size` は `mm×mm` 形式を維持する（並び順ロジックに使用する。作品一覧の画像枠比率は実画像の縦横比を優先する）
- 画像複数枚時は1枚目を「余白のない画面のみ」から選び、2枚目を「余白ありかつ歪み・傾き最小」で選ぶ
- 画像を追加・差し替えたら `image` と `images[0]` の整合を必ず確認する
- 作品サイズの `size` は `NNNmm×NNNmm` 形式で統一する（`mm` 省略を禁止）
- 作品画像を追加したら、公開前に `node scripts/prepare-work-images.js <画像パス...>` を実行し、長辺をWeb向け上限に自動調整する（拡大はしない）
- 作品画像を追加・差し替えたら、一覧表示用に `node scripts/generate-list-thumbnails.js` を実行し、`assets/works/list/` の長辺800px派生画像を更新する
- 作品画像の原本は `/Users/IHEI1/展示関係/portfolio-img` を正本とし、Web側では原本を直接使わない
- 作品画像をWebサイトへ取り込む時は、`node scripts/import-work-image.js <sourcePath> <destFileName>` を使い、`assets/works` へコピー・リネーム・最適化してから使用する
- コピー元が `/Users/IHEI1/展示関係/portfolio-img` 配下の場合、原本は絶対に削除・移動・上書きしない（常に「コピーのみ」）
- `data.js` で参照する画像パスは、`assets/works` 内に実ファイルが存在する名前のみを使う（`-v2` 等の枝番は、実ファイル作成後にのみ参照へ設定する）
- 差し替えで不要になった画像は、完全削除せず `~/.Trash/` 配下へ移動して保管する（即時復旧できる状態を維持）
- i18n 文言追加時は `data-i18n` / `data-i18n-html` を設定し、`i18n.js` の `ja/en` 両方に同一キーを追加する
- ページ追加時は `<title data-i18n="page_title_*">` と `<h1>` を必須とする
- 下位カテゴリを持つカテゴリ（版画/デジタル/漫画）は、全ページのヘッダーで同じドロップダウン構造を維持し、各カテゴリページの `intro` 内に控えめな下位カテゴリ導線を置く
- 下位カテゴリページのタイトル・H1・intro導線・ナビ/リスト表記は、同じカテゴリ名を使って表記ゆれを避ける
- デジタルは独立した親ページ（`digital.html`）を置かず、デジタル導線の既定リンク先は `digital-illustration.html` とする
- styles のブレークポイントは `760px` と `520px` を基準とし、既存の余白・文字スケールを優先して拡張する
- 作品一覧では、作品画像の全体表示・余白・整然とした配置を優先する。版画/デジタル等の通常作品は共通の鑑賞枠を基本にし、画像本体を中央配置して `object-fit: contain` で歪みや切り抜きを避ける
- 作品一覧は詳細ページへの導線であり、詳細鑑賞の本番ではない。通常作品では細部視認よりも、作品同士が干渉しない余白・整列・全体像の把握を優先する
- 作品一覧の `<img>` は `assets/works/list/` の軽量派生画像を使い、詳細ページ・ビューアでは `assets/works/` の元画像を使う
- PCの通常作品一覧は3列基準とし、列間と画像内側余白を十分に取る。画像枠・画像本体はカード幅内に必ず収め、隣のカードへはみ出させない
- PCの通常作品一覧で横比率の大きい作品が小さく見えすぎる場合は、3作品ぶんの行を2作品で分け合う幅にしてよい。実装上は3列を内部6分割し、通常作品を2枠、横長作品を3枠にする。横長作品だけが2作品ぶんを占有して目立ちすぎる配置は避ける
- 作品比率がばらつく一覧では、カード位置は上から整列し、画像本体は共通の鑑賞枠内で中央配置する。作品ごとの実比率で枠の高さを大きく揺らすより、共通枠と余白で一覧の秩序を作る
- 極端な縦長/横長画像では、枠比率に上下限を設けて余白を保ちながら全体を縮小表示する。固定高さで切り抜いて整える実装は通常作品では避ける
- 四コマ漫画の一覧のみ、一覧上の入口として拡大トリミング表示を許容する。本文確認は詳細/ビューアで行う前提とする
- 四コマ漫画の姉妹シリーズは `series: "sisters"` と `seriesOrder` で管理する。`seriesOrder` は一覧の表示順切替（第一話から/最新話から）に使うが、一覧タイトルには話数を表示しない
- レイアウト崩れ対策で `overflow-x: hidden` だけに依存しない。グリッド列幅、画像枠、画像本体の実寸が親要素内に収まっていることを検証する
- `_headers` と `_redirects` は公開設定に関わるため、削除や移動はしない（整理用コピーは `hosting-config/` を参照）
- 内部リンク（`*.html`）はHTMLの相対リンクを正とし、JSで一括書き換えしない
- `_redirects` は公開環境での実挙動を確認しない限り拡張しない（不要なrewrite追加を禁止）
- 作品情報を追加・修正したら、`data.js` を正本として更新し、`node scripts/generate-work-pages.js` でルート直下の `work-*.html` 詳細ページを再生成する（生成物は個別手編集しない）

## PC / モバイル構成差分ルール（維持必須）
- レイアウト分岐の基準は `760px` / `520px` とし、`app.js` の `isMobile: (max-width: 760px)` 判定と `styles.css` のメディアクエリを一致させる
- `760px` 以下（モバイル）では、全体レイアウトを基本1カラム基調に切り替える。ただし作品一覧（`data-gallery-layout="compact"`）は作品の比較性を優先し、基本は横に3作品並べる
- `760px` 以下の一覧（`data-gallery-layout="compact"`）は、通常作品では共通枠内での全体表示を維持する。基本は横に3作品表示とし、横長作品がある場合は3作品ぶんの行を2作品で分け合う幅にしてよい
- 銅版画一覧では、横比率の大きい作品に `is-wide-landscape` が自動付与された場合、3作品ぶんの行を2作品で分け合う幅にする。ただし画像本体は左右余白で抑え、他作品より強く主張しすぎない表示にする
- `760px` 以下では一覧（`compact`）の詳細情報を抑制する（`caption-meta-list` / `caption-text` / `caption-toggle` 非表示）
- `520px` 以下でも、作品一覧は横3作品基本を維持する。古い「カード列を1列化する」方針は作品一覧には適用しない
- 作品一覧は表示サイズ選択UIを置かず、常に `compact` 相当の一覧表示に固定する
- 作品一覧ページには表示順切替（デフォルト / 制作年度順 / 作品サイズ順）を置き、TOPの Selected Works には適用しない
- 画像ビューア表示中は言語切替ボタンを非表示にする（重なり防止のため `body.viewer-open .lang-switch` を維持）
- 作品詳細ページの「作品一覧へ」は、作品詳細に入る直前にいた一覧ページURLへ戻す（直接アクセス時は作品カテゴリの一覧へ戻す）
- モバイルの横スクロール誘導は同じ位置にバーを重複表示しない。OS標準バーの色指定が効かない場合は、標準バーを非表示にして操作可能なサイト側バーへ置換する

## トップヒーロー運用ルール（再現性必須）
- ヒーロータイトルは `data-i18n-html="top_hero_title"` を使う（HTML入り文言のため `data-i18n` は使わない）
- トップの切替画像は常に5枚で運用する
- 5枚のうち2枚は「木版画かつ1000mm超」から選出する
- 切替画像の候補選出はページ読み込みごとにランダム再抽選する
- 1000mm超判定は `size` の数値（幅または高さ）で行う

## 公開前チェック
- `data.js` 内の `image` / `images[]` 参照先ファイルが1件も欠落していないことを確認する
- 画像差し替えを含む公開では、ローカル作業コピーと公開リポジトリの対象画像SHA-1を照合して一致させる
- 全ページでナビリンク・言語切替・ギャラリー描画・画像ビューアが動作する
- 作品キャプションの折りたたみ（続きを読む/閉じる）が機能する
- i18n切替後に title/見出し/導線文言が欠落しない
- 主要画像に不正な向き・欠落がない
- TOPから各ページ（版画/木版画/銅版画/デジタル/漫画/Profile/販売）への遷移が公開URLで成功する
