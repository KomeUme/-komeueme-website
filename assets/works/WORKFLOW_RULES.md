# 画像差し替え運用ルール（再現性優先）

## 1. 作業場所の原則
- 作業用コピーと GitHub 管理リポジトリは別物として扱う。
- 反映対象が GitHub の場合、最終更新は必ず `/Users/IHEI1/Documents/GitHub/-komeueme-website` 側で行う。

## 2. 作品画像の差し替え手順（夕暮れのような複数枚作品）
1. `data.js` で対象作品タイトルと画像パスを特定する。
2. 2枚目など差し替え対象ファイル名（例: `assets/works/work-19-02.jpeg`）を確定する。
3. 新画像をそのファイル名に上書きする（拡張子も一致させる）。
4. `ls -l` でサイズ・更新時刻が変わったことを確認する。
5. 必要なら `shasum` で作業元と GitHub 側の一致を確認する。

## 3. 拡張子のルール
- `.jpg` と `.jpeg` は別ファイルとして扱う。
- `data.js` の参照拡張子と、実ファイル名の拡張子を必ず一致させる。
- 参照だけ変えて実体が古い、または実体だけ変えて参照が古い、の不整合を禁止する。

## 4. GitHub 反映の最小チェックリスト
1. `git status --short` で差分確認。
2. 画像差し替え時は `ls -l`（必要なら `shasum`）で実体確認。
3. `git add` → `git commit` → `git push` を同一リポジトリで実行。
4. push 後にコミットIDを記録する。

## 5. 表示未更新に見える場合の切り分け順
1. GitHub リポジトリ内ファイルサイズ・更新時刻を確認。
2. `data.js` の参照先が想定どおりか確認。
3. ブラウザをハードリロードしてキャッシュ影響を除外。
